# Dataset Card: `reviews`

Following the spirit of [Datasheets for Datasets](https://arxiv.org/abs/1803.09010) (Gebru et al., 2018).

## Summary

A fully synthetic set of ~2,200 product reviews for the [products dataset](../products/DATASET_CARD.md), generated by combining a per-category aspect/sentiment phrase bank with a seeded random number generator. **No real reviewers, review text, or scraped content are used anywhere in this dataset.**

It exists so GlassCart's reviews subsystem has realistic-shaped review text to run sentiment scoring, aspect extraction, and fake-review detection against — none of which is possible against the product dataset alone, since that dataset only carries an aggregate `rating`/`rating_count` per product, no review text.

## Schema

| Field                | Type    | Description                                                        |
|----------------------|---------|----------------------------------------------------------------------|
| `id`                 | string  | Stable review id, e.g. `RP00042-03`                                  |
| `product_id`         | string  | Id of the reviewed product (see `products.json`)                     |
| `author`             | string  | Synthetic reviewer name                                              |
| `rating`             | int     | 1–5 stars, scattered around the product's aggregate rating           |
| `title`              | string  | Short review title                                                    |
| `body`               | string  | Review text: an opening sentence, 1–3 aspect-sentiment sentences, a closing sentence |
| `verified_purchase`  | bool    | ~85% true for genuine reviews, always false for synthetic fakes       |
| `helpful_votes`      | int     | 0–120 for genuine reviews, 0–3 for synthetic fakes                    |
| `created_at`         | string  | ISO date, 2024–2026                                                  |
| `is_fake_synthetic`  | bool    | **Ground-truth label, not a model output.** True for the ~8% of reviews deliberately generated to look like spam/bot reviews. Used only to *evaluate* the fake-review heuristic in [`training/reviews/analyze.py`](../../training/reviews/analyze.py) — the heuristic itself never reads this field, only the review's surface features (see the [model card](../../models/reviews/MODEL_CARD.md)). |

## Provenance & Generation Process

Generated entirely by [`generate.py`](generate.py) with a fixed seed (`20260704`), so **the dataset is byte-for-byte reproducible**:

```bash
uv run datasets/reviews/generate.py
```

The generator:
1. Defines a small aspect vocabulary per category (e.g. Electronics: battery life, build quality, connectivity, price), each aspect with a bank of positive/negative phrases.
2. For each product, generates 2–11 reviews. Each review's star rating scatters around the product's own aggregate rating (a triangular distribution, not a copy of it — individual reviews are noisier than their own average), and its body mentions 1–3 aspects with sentiment matching the star rating.
3. Separately, injects a small number of synthetic "fake" reviews: a handful of synthetic bot authors each post the same generic, superlative-only text (no aspect mentions) to several unrelated products within a tight date range — the classic bot-spam signature the fake-review heuristic is built to catch. These are the only reviews with `is_fake_synthetic: true`.

No network access, no external API keys, and no manual curation are required.

## Intended Use

- Input corpus for the reviews subsystem: sentiment scoring, per-aspect sentiment extraction, and fake-review heuristic detection (see [docs/subsystems/reviews.md](../../docs/subsystems/reviews.md)).
- A **held-out evaluation set** for the fake-review heuristic specifically, via the `is_fake_synthetic` ground-truth field — see the [model card](../../models/reviews/MODEL_CARD.md)'s Metrics section for the honest caveat about evaluating a heuristic against the same generator's own synthetic labels.
- Teaching example for building a synthetic dataset with a *deliberately engineered* signal (aspect-sentiment structure, a documented bot-spam pattern) rather than pure noise, so a downstream heuristic has something real to find.

## Non-Intended Use

- Not intended to represent real customer opinions, real product quality, or real review-fraud patterns. Do not use this dataset to draw conclusions about any real product or seller.
- Not a benchmark dataset for academic sentiment-analysis or fake-review-detection research — the fake-review pattern here is a simple, hand-authored bot signature, not representative of the sophistication of real review fraud.

## Limitations & Bias Discussion

- **Templated language**: review bodies are built from a fixed sentence-slot structure and a small phrase bank per aspect, so lexical diversity is far lower than real reviews — sentiment scoring will look artificially easy compared to production data.
- **Aspect vocabulary is hand-authored** by the GlassCart maintainers, not sourced from real customer feedback, and is deliberately small (3–5 aspects per category).
- **The fake-review pattern is simplistic and known in advance**: real bot/spam reviews are far more varied and adversarial. A heuristic that performs well here should not be assumed to generalize to real fraud detection.
- **Review volume is modest** (~2,200 reviews across 312 products, 2–11 per product) — too small for any aggregate statistic per product to be highly reliable.

## Regeneration / Extension Instructions

To change the aspect vocabulary, sentiment phrase banks, review volume, or fake-review rate, edit the constants at the top of [`generate.py`](generate.py) and re-run:

```bash
uv run datasets/products/generate.py       # if the product catalog itself changed
uv run datasets/reviews/generate.py
uv run training/reviews/analyze.py         # rebuild sentiment/aspect/fake-review artifacts
```
